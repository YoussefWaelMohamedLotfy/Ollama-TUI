# Security Policy

## Supported Versions

The following versions of Ollama-TUI receive security fixes:

| Version | Supported |
| ------- | --------- |
| Latest (`main`) | ✅ |
| Older releases  | ❌ |

## Reporting a Vulnerability

We take the security of Ollama-TUI seriously. If you discover a security vulnerability, **please do not open a public GitHub issue**.

### Option 1 — GitHub Private Vulnerability Reporting (Preferred)

Use GitHub's built-in private reporting feature:

1. Go to the [Security Advisories](https://github.com/YoussefWaelMohamedLotfy/Ollama-TUI/security/advisories/new) page.
2. Click **"Report a vulnerability"** and fill in the details.
3. Your report will be visible only to the maintainers until it is resolved and publicly disclosed.

### Option 2 — Email

If you prefer email, send your report privately to:

**youssefwael8@gmail.com**

Please use the subject line `[SECURITY] Ollama-TUI Vulnerability Report`.

You can also reach the maintainer via the [GitHub profile](https://github.com/YoussefWaelMohamedLotfy).

### What to Include

Please provide as much detail as possible in your report:

- A clear description of the vulnerability and its potential impact.
- Steps to reproduce the issue (proof-of-concept code, screenshots, or logs).
- The affected version(s) or commit SHA.
- Any suggested mitigation or fix if you have one.

## Response Process

1. **Acknowledgement** — We will acknowledge receipt of your report within **5 business days**.
2. **Triage** — We will assess severity and reproduce the issue within **10 business days**.
3. **Resolution** — We will work on a fix and keep you informed of progress throughout.
4. **Public Disclosure** — Once a fix is released, we will coordinate with you on a responsible public disclosure timeline. We aim to disclose within **90 days** of the initial report.

## Responsible Disclosure

We kindly ask that you:

- Give us a reasonable time to triage and patch the vulnerability before any public disclosure.
- Avoid accessing, modifying, or deleting user data during your research.
- Act in good faith and avoid causing harm to the project or its users.

We are committed to working with security researchers and will credit reporters in our release notes (unless you prefer to remain anonymous).

## Scope

This policy applies to security vulnerabilities in the Ollama-TUI source code found in this repository. Vulnerabilities in third-party dependencies (e.g., OllamaSharp, ModelContextProtocol, Spectre.Console) should be reported to their respective maintainers.
